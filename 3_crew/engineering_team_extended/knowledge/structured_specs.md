## Structured Module Specification Schema

The engineering lead must respond with a single JSON object that adheres to the schema below. All string values should be plain text (no markdown). Numeric values must be numbers, not strings.

```json
{
  "system_name": "string",
  "overview": "string",
  "modules": [
    {
      "filename": "string",
      "class_name": "string",
      "responsibility": "string",
      "dependencies": ["string", "..."],
      "public_api": [
        {
          "name": "string",
          "signature": "string",
          "description": "string",
          "inputs": [
            {
              "name": "string",
              "type": "string",
              "description": "string"
            }
          ],
          "outputs": [
            {
              "type": "string",
              "description": "string"
            }
          ]
        }
      ]
    }
  ]
}
```

### Field Notes

- `system_name`: Human-readable title for the solution (e.g., `"Trading Portfolio Suite"`).
- `overview`: High-level narrative explaining how the modules collaborate.
- `modules`: Each entry defines one Python module to be generated by downstream agents.
  - `filename`: Relative filename (e.g., `"accounts.py"`). Must be unique.
  - `class_name`: Main class that anchors the module.
  - `responsibility`: Concise statement of what the module owns.
  - `dependencies`: Optional list of other module filenames this module relies on.
  - `public_api`: Ordered list of functions or methods that form the contract.
    - `signature`: Full Python signature including type hints.
    - `inputs`/`outputs`: Use to clarify expectations so implementers can validate success.

### Example

```json
{
  "system_name": "Trading Portfolio Suite",
  "overview": "Handles account cash, portfolio valuation, and reporting.",
  "modules": [
    {
      "filename": "accounts.py",
      "class_name": "AccountLedger",
      "responsibility": "Manage cash balances and enforce trading rules.",
      "dependencies": [],
      "public_api": [
        {
          "name": "deposit",
          "signature": "def deposit(self, amount: float) -> bool",
          "description": "Add funds if amount > 0.",
          "inputs": [
            {
              "name": "amount",
              "type": "float",
              "description": "Cash to deposit"
            }
          ],
          "outputs": [
            { "type": "bool", "description": "True when deposit succeeds" }
          ]
        }
      ]
    }
  ]
}
```

Downstream agents should treat missing optional arrays as empty lists.
